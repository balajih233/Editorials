Two Sum

The given problem says that there are n numbers present in an array and a target. We need to find two numbers which sums up target and return indexes of two numbers.


Let us take an example of an array arr[] = [4,3,9,6,2] and a target element 11.


From this array elements we can say that index at 2 and index at 4 sums up to 11. arr[2] + arr[4] = 11. So we need to return indexes 2 and 4.


If we need to return boolean value then we can do this with two pointers approach. But the given solution should return indexes. So I will take a Hash Map and store the indexes of each element. Keys of HashMap are elements and values are indexes.


I will iterate through array and check whether difference between array element and target is exit in map or not. If the difference is present then I will return current index and difference's index.


At index 0, element is 4 is present. I will calculate difference between element(4) and target(11) which is 7. Currently, My HashMap is empty so i will add element 4 and value as 0.

Time Complexity : O(n)

Space Complexity : O(n)

Java Code: 
            
class Solution {
    public int[] twoSum(int[] nums, int target) {
        int ans[]=new int[2];
        HashMap<Integer,Integer> x=new HashMap<>();
        for(int i=0;i<nums.length;i++){
            int diff = target - nums[i];
            if(x.containsKey(diff)){
                ans[0] = x.get(diff);
                ans[1] = i;
                return ans;
            }
            x.put(nums[i],i);
        }
         return -1;
    }
}

Palindrome Number

There is an integer x and we need to check whether it is Palindrome or not. A Palindrome number is a number which reads same from left to right.

Take an example x=147741. If I reverse the number then it remains same. If the number is 451 and if I reverse it then it becomes 154 which is not equal to given integer. If the number is negative then it is not a Palindrome. For example, x=-123. If I reverse it then it becomes 123- which is not equal to given number. We can solve this problem in different methods.

Approach-1

In this approach, I will store integer x in another variable. If the value is less than 0 then I will return false. If the value is greater than or equal to zero and less than 10 then I will return true. Otherwise, I store the value in another variable. Now I will reverse the number. If reveresed number is equal to original number then I will return true otherwise I will return false.

I copy the number into a variable called originalNumber. I take a reversedNumber variable and assign 0 to it. To perform this, I will calculate remainder and multiply reversedNumber with 10 and add remainder to it. I will divide x with 10. I will perform this operation until my given number 'x' is equal to zero.

Let us take an example, x = 4554. I will copy x into variable originalNumber and initialize reversedNumber as 0.

  • x = 4554
  • originalNumber = x = 4554.
  • reversedNumber = 0.

Now I will take remainder of x. To know remainder, I will perform modulo operation to x.

  • rem = x % 10 = 4554 % 10 = 4.

Now I will multiply reversedNumber with 10 and add remainder to it. I store it in reversedNumber.

  • reversedNumber = (reversedNumber * 10) + rem => (0 * 10) + 4 = 4.
  • reversedNumber becomes 4.

I will divide integer x with 10 then x becomes 455.

  • x= 4554 / 10 = 455.

My x is not zero so, I will repeat the above procedure.

Now I will take remainder of x. To know remainder, I will perform modulo operation to x.

  • rem = x % 10 = 455 % 10 = 5.

Now I will multiply reversedNumber with 10 and add remainder to it. I store it in reversedNumber.

  • reversedNumber = (reversedNumber * 10) + rem => (4 * 10) + 5 = 40 + 5 = 45.
  • reversedNumber becomes 45.

I will divide integer x with 10 then x becomes 45.

  • x= 455 / 10 = 45.

My x is not zero so, I will repeat the above procedure.

Now I will take remainder of x. To know remainder, I will perform modulo operation to x.

  • rem = x % 10 = 45 % 10 = 5.

Now I will multiply reversedNumber with 10 and add remainder to it. I store it in reversedNumber.

  • reversedNumber = (reversedNumber * 10) + rem => (45 * 10) + 5 = 450 + 5 = 455.
  • reversedNumber becomes 455.

I will divide integer x with 10 then x becomes 4.

  • x= 45 / 10 = 4.

My x is not zero so, I will repeat the above procedure.

Now I will take remainder of x. To know remainder, I will perform modulo operation to x.

  • rem = x % 10 = 4 % 10 = 4.

Now I will multiply reversedNumber with 10 and add remainder to it. I store it in reversedNumber.

  • reversedNumber = (reversedNumber * 10) + rem => (455 * 10) + 4 = 4550 + 4 = 4554.
  • reversedNumber becomes 455.

I will divide integer x with 10 then x becomes 4.

  • x= 45 / 10 = 4.

My x is zero so, I will check whether my reversedNumber is equal to originalNumber reversedNumber is 4554 and originalNumber is 4554 this indicates both are equal. Hence given number is a Palindrome number.

class Solution {
    public boolean isPalindrome(int x) {
        if(x < 0) return false;
        if(x < 10) return true;
        int originalNumber = x;
        int reversedNumber = 0;
        while(x!=0){
            int rem = x%10;
            reversedNumber = (reversedNumber * 10) + rem;
            x = x / 10;
        }
        if(originalNumber == reversedNumber) return true;
        return false;
    }
}

Approach-2

In this approach, I will typecast integer element to string and reverse the string. I will check whether reversed string is equal to original string.

class Solution {
    public boolean isPalindrome(int x) {
        if(x < 0) return false;
        if(x < 10) return true;
        String s=Integer.toString(x);
        String reverse="";
        for(int i=0;i < s.length();i++){
            reverse = s.charAt(i) + reverse;
        }
        return s.equals(reverse);
    }
}

Happy Number

A number is said to be happy if sum of squares of digits is equal to 1. The process must be repeated until the number equal to 1.

Consider a number num = 19.

  • (1*1) + (9*9) = 82
  • (8*8) + (2*2) = 68
  • (6*6) + (8*8) = 100
  • (1*1) + (0*0) + (0*0) = 1

Here num is equal to 1. Hence number 19 is a happy number.

In this problem, I will use recursion. I will do recursion until my num comes to single digit. If the single digit is equal to 1 or 7 then the number is happy-number and I will return true. Otherwise I will return false.

Java Code: 
        
class Solution {
    public boolean isHappy(int n) {
        int sum=0;
        while(n!=0){
            int rem = n % 10;
            sum += (rem * rem);
            n = n /10;
        }
        n=sum;
        if(n==1 || n==7) return true;
        else if(n>10) return isHappy(n);
        else return false;
    }
}

Add Digits

This problem is similar to happy number. We need to sum up the digits in a number until it becomes single digit.

Consider an example number num=654

  • 654 -> 6 + 5 + 4 = 15
  • 15 -> 1 + 5 =6.

Here 6 is a single digit and I return this number.

I will use recursion to solve this problem. First I sum up each digit in number. I will divide the sum with 10 and check whether the quotient is 0 or not. If the quotient is 0 then I will return sum otherwise I will call my function again.

                class Solution {
                    public int[][] sortMatrix(int[][] grid) {
                        int mat[][]=new int[grid.length][grid[0].length];
                        int r=grid.length-1;
                        int c=0;
                        while(r>=0 && c<grid[0].length){
                            int i=r;
                            int j=0;
                            List<Integer>x=new ArrayList<>();
                            while(i<grid.length && j<=c){
                                x.add(grid[i][j]);
                                i++;
                                j++;
                            }
                            Collections.sort(x);
                            i=r;
                            j=0;
                            int k=x.size()-1;
                            while(i<grid.length && j<=c){
                                mat[i][j]=x.get(k--);
                                i++;
                                j++;
                            }
                            r--;
                            c++;
                        }
                        r=0;
                        c=grid[0].length-1;
                        while(r<grid.length && c>=0){
                            int i=0;
                            int j=c;
                            List<Integer>x=new ArrayList<>();
                            while(i<=r && j<grid[0].length){

                                x.add(grid[i][j]);
                                i++;
                                j++;
                            }
                            Collections.sort(x);
                            i=0;
                            j=c;
                            int k=0;
                            while(i&=r && j<grid[0].length){
                                mat[i][j]=x.get(k++);
                                i++;
                                j++;
                            }
                            r++;
                            c--;
                            if(r==grid.length-1) break;
                        }
                        return mat;
                    }
                }

class Solution:
    def sortMatrix(self, grid: List[List[int]]) -> List[List[int]]:
        mat = [[0] * len(grid[0]) for _ in range(len(grid))]
        r = len(grid) - 1

        c = 0
                
        while r >= 0 and c < len(grid[0]):
            i = r
            j = 0
            x = []
            while i < len(grid) and j <= c:

                i += 1
                j += 1
            x.sort()
            i = r
            j = 0
            k = len(x) - 1
            while i < len(grid) and j <= c:
                mat[i][j] = x[k]
                k -= 1
                i += 1
                j += 1
            r -= 1
            c += 1
